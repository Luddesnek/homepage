1.0.5. What will you post here

My thoughts, or something cool i want to share like this script i made to play music 

#!/bin/bash

x=$(ls /home/lu/Fatty/Music/ | fzf)
y=$(ls /home/lu/Fatty/Music/"$x" | fzf)

mpv --no-video /home/lu/Fatty/"$x"/"$y"

It just opens my folders with fzf and chooses a item then plays it in mpv.

It's really nothing special but i found it really useful. It's interesting that you can make something so useful. with just three lines of "code".

Also ignore my folder name. Fatty is what i call my HDD.

To be honest. It looks worse today, probably the best colors i have had, I didn't think so at the time i had them.

One thing i want to mention is the blocks in the right corner the left monitor. They are basic one-line scripts and one thing i noticed as newbie that there is a looot of way to write these scripts.

Example: 

sensors | tr ' '+ ' ' | awk '/Package/'\
	| awk '{print "[ "$4}' | cut -b 1-4,7-8

There is like million ways to write a script like this. The thing i need the script to do is diplay my cpu temperature like this: [ 40°

that script is actually pretty stupid, because you use awk too times, and tr is used when you can just use cut -b, because cut is already in use. 

sensors | awk '/Pa/ {print "[ " $4}'\
	| cut -b 1-2,4,5,8-9

↑ that is what i came up with right now and there is probably other ways to make it even better. Some linux expert would probably kill me for using this. 

sensors | grep "Package" | sed s/+//g | awk '{print $4}'

↑ that is the first version of the script and i don't even want to talk about it.

To be fair i'm just starting to learn shell scripting and i don't know if everything above even counts as shell scripting.
